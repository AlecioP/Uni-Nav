KEY VALUE
Nome : prenota(Partenza,Arrivo,Navetta,Giro)
Riferimenti : Gestione Prenotazioni
Pre-condizioni :
  • Il sistema deve conoscere la matricola dello studente che sta effettuando la richiesta. Per questo motivo lo studente dovrà già essere autenticato
  • Il sistema deve conoscere la data odierna
  • Il sistema deve conoscere l'identitá dell'Autista alla guida della navetta
Post-condizioni :
  • Viene creata un'istanza di Prenotazione
  • Viene creata un'associazione tra la nuova Prenotazione e lo Studente prenotante
  • Viene creata un'istanza di Biglietto
  • Viene creata un'associazione tra il nuovo Biglietto e la nuova Prenotazione
  • Viene creata una o piú associazioni tra la nuova Prenotazione ed uno o piú Tratti di linea
  • Viene creata un'associazione tra la nuova Prenotazione ed il Registro delle Prenotazioni
  • Viene creata un'associazione tra la nuova Prenotazione ed una istanza di Navetta
